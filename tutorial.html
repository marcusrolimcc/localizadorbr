Para iniciantes no mundo da programação ou calouros de ciência da computação

INTRODUÇÃO

Este tutorial tem o intuito de instruir todo desenvolvedor iniciante criar esse site de serviço, utilizando uma API externa, trabalhando com as ferramentas de front-end.

Iniciaremos com a utilização do HTML, para estruturar o site, assim utilizaremos o JavaScript para manipularmos dados e por fim vamos estilizar o site utilizando apenas o BootStrap 4.

É recomendado que tenha conhecimento básico de computação para concluir o desenvolvimento, porém será disponibilizado links de material de apoio para estudar e adquirir o conhecimento necessário para a conclusão do projeto.

DESENVOLVIMENTO LOCALIZADORBR

O site de serviço tem como objetivo utilizar uma API, disponibilizada pelo IBGE, onde pode se fazer uma busca das regiões do Brasil, estados, cidades além das regiões meso, micro, todos os dados automaticamente atualizados diretamente do site do órgão governamental.

Iniciando o desenvolvimento, é necessário a utilização de uma IDE.

A IDE utilizada para o desenvolvimento do projeto foi o Visual Studio, caso não tenha um hardware com bom desempenho pode se utilizar outras IDEs como Sublime Text que também é uma IDE profissional, ou pode ser utilizada a universitária Code Blocks.

Estando dentro da IDE escolhida, iniciamos os primeiros passos.

4º PASSO - Acessando os dados:

Para acessarmos os dados, será necessário criar um arquivo JavaScript, que é a ferramenta de linguagem de programação do front-end.

Logo iniciando o arquivo JavaScript, deve se executar funções, através de alguns comandos.

O $ é chamado de seletor, ele é a mão do JavaScript Jquery, assim inicia a função, assim logo que se abre o site é executado as funções iniciais.

Estados();

O processo é simples, a função tem o objetivo de buscar os dados da API, no caso a função busca os dados em um link que o IBGE disponibiliza.

Para isso, cria-se uma variável (var) que se atribui o endereço da API, em seguida, acessa o endereço e pega ($.get) a variável e executa outra função, de retorno e de status.

O retorno basicamente é a resposta do que achou no endereço e o status é a mensagem que o servidor entrega, via HTTP/HTTPS.

AÇÃO COMPLEMENTAR:

Para conseguir acessar os dados da API, e entender como ela está estruturada, para poder saber como fazer a pesquisa por ela, utilizamos uma ferramenta chamada POSTMAN.

Onde se insere o endereço que quer acessar, e essa ferramenta mostra, como está estruturado.



Assim visualizando a forma de estrutura, podemos verificar que se inicia com [, fazendo assim procurarmos um retorno de um array, que são matrizes que armazenam vários valores em uma única variável, assim, verificamos quais são os valores dentro do array e as nomes de suas classes.

Entendido como acessamos os arquivos de dados da API. Vamos continuar com a resposta, o retorno do servidor.

Com esse retorno temos duas opções, caso não encontre nada, ou caso encontremos algo.

Assim executamos uma condicional, onde se a resposta do retorno for negativa, (if retorno.length == 0) , traduzindo o comando, .length mostra o número de caracteres, no caso strings tem, caso não tenha nenhuma string, a lógica é que não tenha encontrado nada, fazendo assim.

Uma mensagem de erro para o usuário, através do comando alert, adicionando o status, para conseguir identificar qual foi o problema ocorrido, através do protocolo enviado pelo servidor.

Caso encontre algum dado no endereço, temos a segunda opção, (else) onde iniciamos com a função show(false) que será desenvolvida no final da aplicação.

E assim, ligamos o nosso primeiro bloco do HTML, que chamamos de “estados” com o comando (‘#estados’) onde o # é o indicador que está ligado à alguma coisa, fazendo assim a continuação do comando .append que atribui um valor booleano (falso, verdadeiro; 0 ou 1) para agir o próximo passo que é utilizar o banco de opções que o usuário poderá escolher, retirada do banco de dados do endereço pesquisado.

Para iniciar atribuímos um valor (value) = 0, onde mostra o texto de “selecionar estado”.

Assim, criamos uma lista encadeada simples, do zero ao fim, de todos os dados do endereço acessado, utilizando uma estrutura de repetição (for), onde criamos o método de de 0 ao fim.

(var i = 0; i < retorno.length; i++)

Cria uma variável i atribuindo o valor de 0, em seguida faz a condicional para se repetir, sendo que i deve ser sempre menor do que o retorno que se busca, no caso irá buscar todos os dados até não achar mais nada, e assim finalizar a repetição, a forma de caminhar na lista encadeada é toda vez que se acha algum retorno, se incrementa (i++) adicionando um valor de i, fazendo com que o i um momento seja maior que o retorno finalizando as repetições.

Para cada repetição é criada uma variável, no local do valor de i.

var = estado = retorno[i];

Cada vez que essa variável recebe um novo valor de i, ele acessa o bloco HTML, chamado estados e se adiciona os valores do endereço.

$('#estados').append($('option>', { value: estado.id, text: estado.nome }));

Sabendo dessa estrutura da API, então o valor que queremos mostrar nas opções é o id da informação, e se nome, fazendo o comando value: estado.id, isto é dentro de estado, acesse o id e mostre o nome que está cadastrado nesse id, com o text: estado.nome.

Todos essas palavras estão como vemos na estrutura do código da API.

Finalizando essa primeira função.

9º PASSO - A função Show, para criar uma estética agradável para o usuário.

Ainda no arquivo JavaScript, onde desde o início adicionamos a função Show(false), vamos entendê-la.

Basicamente é uma condicional, para mostrar ou não mostrar os detalhes das cidades escolhidas, esse artifício funciona da seguinte forma.

Se cria uma condicional, if/else, onde se utiliza uma condição booleana, true/false, onde caso a função seja atribuída um valor de true, ela mostrará o bloco chamado #detalhes, caso seja atribuído o valor de false, ela desaparecerá.

Tudo isso pelos comandos .show (mostrar) e o .hide (esconder).

Assim toda vez, que mudarmos de estado, as informações da cidade do estado escolhido anteriormente sumam, “limpando” a tela de informações “antigas”.

Para finalizar o site, criamos um rodapé onde de costume, utiliza-se para mostrar contatos, ou links para informações complementares.

A funcionalidade da row é simples, divide qualquer tela, em 12 partes iguais, e assim, pode ser dividido em cada dispositivo de alguma forma.

Como em nosso rodapé, criamos 2 blocos, onde um deles, no caso o bloco com mais textos e botões, utilizou 8 partes dos 12 com o comando col-sm-8, fazendo assim quando o dispositivo for maior que um small device, no caso um celular, ele ocupará 8 unidades das 12.

Já o outro bloco que utiliza a imagem do logo tipo do IBGE, utiliza apenas 4 espaços, totalizando os 12. Com o comando col-sm-4.

Em cada bloco utilizamos comandos HTML diferentes, no primeiro bloco, utilizamos um botão para acessar esse tutorial.

O comando button> onde escolhemos o tipo dele ser um botão, e a classe do bootstrap que utilizamos é a de botão através do “btn” e adicionei uma cor amarela com o comando btn-warning.

E o botão contém o conteúdo de um link através do comando a>, onde colocamos o “href” onde ele se refere ao endereço, em nosso caso, o tutorial.html.

Já no segundo bloco, utilizamos um link contendo uma imagem, do logo do IBGE, fazendo com que o link se torne a imagem, uma forma diferente visualmente de se acessar um link.
